Getting started

Let’s start from the following functions:

R’s pipe is different from the original F#’s pipe. R’s pipe isn’t only flip (id) but also provides the functionality of partial application. The function (%<%) plays a role of partial application. We visualize (%<%) as follows: [A⊗(B⊗C),D]→curry[A,[B⊗C,D]]→[1A,curry][A,[B,[C,D]]]↓[α,1D]def↓ϕ∥[(A⊗B)⊗C,D]→curry[A⊗B,[C,D]]→curry[A,[B,[C,D]]], where [A,B] is a set of maps from a set A to a set B, 1X:X→X is the identity map on a set X, α is the associator with respect to the tensor product (⊗) of monoidal category. The tensor product (⊗) corresponds to pairlist of R. The function (%<%) is given by (%<%) := phi (id).

Let’s consider the associativity of addition (1+2)+3=1+(2+3). This equation is written in the point-free style as (+)∘((+)⊗id)=(+)∘(id⊗(+))∘α, where α is the associator. The associator is highly related to the associativity.

Since an argument list of function is not a first-class citizen in almost programming languages, we can’t directly define the associator as a function in codes. The partial application (%<%) can be regarded as a substitution of the associator.

To make the functionality of (%<%) visible, we’d like to play with monoidal functors. One of the easiest ways to make a monoidal functor is a monad, because, on the category of sets, every monad is an monoidal functor.

map_2 of reader_monad and list_monad work as follows:

  • reader_monad: reader-map_2
    Many algebraic structure on real numbers are straight-forwardly translated to map targeting real numbers, e.g. (f+g)(x):=f(x)+g(x) for all f,g:X→R with some set X. map_2 of reader_monad gives this kind of conventions.
  • list_monad: list-map_2
    It works almost same with outer in R.

There is a question: How to define map_3, map_4 and so on? The partial application %<% helps us to write an ad hoc map_n.

If we define functions like map_2 and pure of monad once, then we obtain map_n automatically by using the partial application (%<%). In this case, (%<%) plays a role to extend map_2 to map_n. This is similar that we usually don’t define the function like \(x, y, z) x + y + z because (+) is enough. In this case, map_2 is enough.
