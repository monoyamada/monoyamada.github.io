Getting started

To be honest, I don’t know Linq. However as far as seeing the example listed in the article, the main gadget of Linq is looked like a list of records.

var results =  from c in SomeCollection
               where c.SomeProperty < 10
               select new {c.SomeProperty, c.OtherProperty};

foreach (var result in results) {
 Console.WriteLine (result);
}

On the other hand, the main gadget of Tidyverse is looked like a spreadsheet, i.e. a record of lists. The correspondence between a list of records and a record of lists is given by

{python, dependson = ""} def none (): AB = collections.namedtuple ("AB", ("a", "b")); tidy = AB (tuple (range (3)), tuple ("abc")); linq = list (map (AB, *tidy)); back = AB (*zip (*linq)); print (tidy, linq, back, sep = "\n"); return None; none ();

This correspondence is fairly similar to the definition of product.

zip_mod :: (Int -> Int -> Int) -> (a -> b -> c) -> [a] -> [b] -> [c]
zip_mod pqr abc as bs = to_list $ zip_reader abc pqr (to_map as) (to_map bs) where
 to_map xs = (\j -> xs !! mod j n, n) where n = length xs
 to_list (ja, n) = [ja j | j <- [0 .. (n - 1)]]

zip_lcm = zip_mod lcm

product_fw :: (Hom x a, Hom x b) -> Hom x (a, b)
product_fw = uncurry $ liftA2 (,)

product_bw :: Hom x (a, b) -> (Hom x a, Hom x b)
product_bw = NonEmpty.unzip

tidy_to_linq :: ([a], [b]) -> [(a, b)]
{-

 -}
tidy_to_linq (as, bs) = zip_lcm (,) as bs

linq_to_tidy :: [(a, b)] -> ([a], [b])
linq_to_tidy abs = tom_to to_list to_list $ product_bw $ to_map abs where
 to_map = (!!)
 to_list ja = map ja [0 .. (n - 1)]
 n = length abs

Monoidal functor is matter

In this section, we consider old plain monoids rather than categorical monoids. A monoid (M,m,u) is characterized by the following two commutative diagrams: (M×M)×Mαm×1M×(M×M)1×mM×MmMM×Mm,I×Mu×1sndI≃M×MmM×I1×ufstI≃M, where

  • (×) is a cartesian product of sets,
  • I is a some fixed singleton set,
    • I is called the unit object of (×).
    • In Haskell, the singleton () is occasionally taken for the unit object.
  • α is the associator of (×),
    • The associator of cartesian product is occasionally omitted.
  • fstI:A×I→A is the restriction of fst:A×B→A.
    • By the restriction, fstI becomes isomorphic.

Let Set:=(Set,⊗,I) be a monoidal category of sets, and (F,μ,η) be a monoidal endo-functor on Set, i.e. F:Set→Set. Let A be a set, then the followings hold: (mtom)(A⊗A)⊗Aαf⊗1A⊗(A⊗A)1⊗fA⊗AgAA⊗Ag⟹(FA⊗FA)⊗FAαμ∗f⊗1FA⊗(FA⊗FA)1⊗μ∗fFA⊗FAμ∗gFAFA⊗FAμ∗g,I⊗Au⊗1sndI≃A⊗AfA⟹I⊗FAη∗u⊗1sndI≃FA⊗FAμ∗fFA, where μ∗ and η∗ are defined as follows: μ∗(A⊗A→hA):=FA⊗FA→μF(A⊗A)→FhFA,η∗(I→vA):=I→ηFI→FvFA. These properties ensure that a monoidal functor (F,μ,η) sends any monoid (M,m,u) to a monoid (FM,μ∗m,η∗u).

By put the associativity diagram onto the diagram that is obtained by acting F to the top-left diagram of (mtom), we have (FA⊗FA)⊗FAαμ⊗1FA⊗(FA⊗FA)1⊗μF(A⊗A)⊗FAμFf⊗1FA⊗F(A⊗A)μ1⊗FfFA⊗FAμF((A⊗A)⊗A)FαF(f⊗1)F(A⊗(A⊗A))F(1⊗f)FA⊗FAμF(A⊗A)FgAF(A⊗A)Fg, where the graph consisting of red lines is the associativity diagram. In a similar way, by putting the unit diagram to the transformed bottom-left diagram of (mtom), we have I⊗FAη⊗1sndI≃FI⊗FAμFu⊗1FA⊗FAμF(I⊗A)F(u⊗1)FsndIF(A⊗A)FfFA.

Note

In general, a functor F:A→B forms a category FA:

  1. an object is Fa for all a∈A,
  2. a morphism is Fa→FfFb for all a→fb∈A.

If A and B are monoidal categories, F is a monoidal functor, then FA fairly reproduce the original monoidal structure of A.
